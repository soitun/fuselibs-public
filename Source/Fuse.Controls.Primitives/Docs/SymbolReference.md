## Symbol Search Tips

### By Function
- **Actions**: Look for verbs like `edit`, `delete`, `share`, `save`
- **Objects**: Look for nouns like `house`, `car`, `phone`, `book`
- **States**: Look for conditions like `on`, `off`, `full`, `empty`
- **Navigation**: Look for directions like `left`, `right`, `up`, `down`

### By Category
Use category prefixes to find related symbols:
- `arrow.*` - Directional symbols
- `person.*` - People-related symbols
- `heart.*` - Love/health symbols
- `house.*` - Home-related symbols
- `clock.*` - Time-related symbols
- `cloud.*` - Weather symbols

### Common Patterns
- `.fill` suffix for filled versions
- `.circle` suffix for circular backgrounds
- `.badge` suffix for notification indicators
- `.slash` suffix for disabled/off states

## Usage Examples

### Basic Usage
```xml
<Image Symbol="heart" />
<Image Symbol="star.fill" />
<Image Symbol="house" />
```

### With Styling
```xml
<Image Symbol="heart.fill" Color="#FF0000" Width="24" Height="24" />
<Image Symbol="star" Color="#FFD700" Margin="4" />
```

### Dynamic Symbols
```xml
<Image Symbol="{isLiked} ? 'heart.fill' : 'heart'" Color="{isLiked} ? '#FF0000' : '#888888'" />
```

## Migration Notes

When migrating from custom icons or other icon systems:

1. **SF Symbols naming**: Use iOS SF Symbol names as the primary key
2. **Material Icons**: Android equivalents are automatically resolved
3. **Fill variants**: Check if your icon has a `.fill` variant available
4. **Size consistency**: System symbols automatically scale appropriately

## Performance Considerations

- Symbol lookup is O(1) dictionary access - very fast
- System symbols are cached by the platform
- Prefer system symbols over custom images when possible
- Use consistent symbol variants throughout your app
